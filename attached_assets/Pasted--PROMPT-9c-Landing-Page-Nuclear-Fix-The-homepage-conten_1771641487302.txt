# PROMPT 9c: Landing Page ‚Äî Nuclear Fix

The homepage content is still not rendering. The root layout is overriding the page styles. Fix this now with the following steps.

## STEP 1 ‚Äî Check what's in app/layout.tsx

Open `app/layout.tsx`. It likely looks something like this:

```tsx
export default function RootLayout({ children }) {
  return (
    <html>
      <body className="some-dark-class">
        <Sidebar />
        <main>{children}</main>
      </body>
    </html>
  )
}
```

The Sidebar and any wrapping divs are hiding or overriding the homepage content.

## STEP 2 ‚Äî Create a route group for the app shell

Restructure the app directory like this:

```
app/
  layout.tsx          ‚Üê root layout (html + body ONLY, no sidebar)
  page.tsx            ‚Üê homepage (standalone, no sidebar)
  pricing/
    page.tsx          ‚Üê standalone page (no sidebar)
  (app)/              ‚Üê route group for sidebar pages
    layout.tsx        ‚Üê sidebar layout (applied to all pages inside)
    lookup/
      page.tsx
    lc-check/
      page.tsx
    dashboard/
      page.tsx
    trades/
      page.tsx
    templates/
      page.tsx
    alerts/
      page.tsx
    demurrage/
      page.tsx
    settings/
      page.tsx
    admin/
      data/
        page.tsx
```

The `(app)` folder name with parentheses is a Next.js route group ‚Äî it does NOT affect the URL. `/lookup` still works. But it gets its own layout (with sidebar) separate from the root.
