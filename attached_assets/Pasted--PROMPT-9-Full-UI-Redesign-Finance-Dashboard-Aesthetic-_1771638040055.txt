# PROMPT 9: Full UI Redesign ‚Äî Finance Dashboard Aesthetic

Apply the following design system to the entire TapTrao application. Every page must be updated: Lookup, LC Checker, Dashboard, Templates, Pricing, Settings, Alerts, Demurrage ‚Äî everything.

Do NOT change any backend logic, database queries, or API routes. This is a visual-only change.

### Nav Items ‚Äî THE ACTIVE STATE LOGIC

This is the core interaction pattern. Whichever page the user is on, that nav item is bright white. Everything else is faded.

```css
.nav-item {
  display: flex;
  align-items: center;
  gap: 8px;
  padding: 7px 8px;
  border-radius: 6px;
  font-size: 13px;
  font-weight: 500;
  color: var(--t2);          /* faded by default */
  cursor: pointer;
  transition: all 0.12s;
}

.nav-item:hover {
  background: rgba(255,255,255,0.05);
  color: var(--t1);          /* brightens on hover */
}

.nav-item.active {
  background: rgba(66,126,255,0.12);
  color: var(--t1);          /* bright white when active */
}

.nav-item.active .nav-icon {
  color: var(--blue);        /* icon turns blue when active */
}

.nav-icon {
  font-size: 14px;
  width: 16px;
  text-align: center;
  color: var(--t3);          /* faded by default */
}
```

**Implementation:** Use Next.js `usePathname()` hook to detect current route. Apply `active` class to the matching nav item automatically. Example:

```tsx

const pathname = usePathname()

<div className={`nav-item ${pathname === '/lookup' ? 'active' : ''}`}>
  <span className="nav-icon">‚ö°</span>

```

## THE WHITE/GREY HIERARCHY ‚Äî APPLIED CONSISTENTLY

This is the most important rule. Apply it to every element across every page:

| Opacity | Usage |
|---------|-------|
| `rgba(255,255,255,0.95)` ‚Äî **var(--t1)** | Page titles, active nav items, card section headers, selected dropdown values, trade route (commodity + countries + arrows), stat numbers, result values |
| `rgba(255,255,255,0.55)` ‚Äî **var(--t2)** | Body text, document names, inactive nav items, general descriptions |
| `rgba(255,255,255,0.28)` ‚Äî **var(--t3)** | Form labels, nav section labels, authority names, timestamps, hash references, stat subtitles, placeholder text, breadcrumbs |

**The rule:** If it IS the data ‚Üí bright white. If it DESCRIBES the data ‚Üí faded.

---

## BORDERS ‚Äî ZERO POLICY

Do not add any border, border-bottom, border-top, border-right, or border-left to any layout element. No lines between sidebar and main. No lines between nav sections. No lines between cards. No lines between document rows. No dividers anywhere.

Use spacing, background colour differences, and border-radius to create visual separation.

The only exceptions permitted:
- `outline: 1px solid var(--blue-bd)` on focused form inputs (functional feedback)
- `border: 1px solid var(--gbd)` on status badges (they are small and need definition)
- The `::before` gradient shimmer on primary cards (this is a glow, not a line)
